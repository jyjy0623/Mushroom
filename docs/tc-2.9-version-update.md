#### TC-2.9-01

**用例名称**：有新版本时弹出更新对话框

**前置条件**：
- GitHub Releases API 返回新版本（如远端 v1.5.0，本地 v1.4.0）
- 有更新日志内容（100 字以内）
- 设备联网正常
- `UPDATE_CHECK_ENABLED = true`

**操作步骤**：
1. 冷启动（完全退出后重新打开）App
2. 等待主界面（每日任务列表）加载完成
3. 等待约 5~10 秒（网络请求时间）
4. 观察是否出现更新对话框

**预期结果**：
1. 主界面正常加载，启动流程无阻塞、无卡顿
2. 主界面加载完成后数秒内，弹出更新对话框（AlertDialog）
3. 对话框标题为「🍄 发现新版本 v1.5.0」
4. 对话框正文显示「当前版本：v1.4.0」
5. 正文中有「更新内容：」标题及具体更新日志文字
6. 对话框底部有两个按钮：「稍后提醒」（左/次级）和「前往下载」（右/主要）

#### TC-2.9-05

**用例名称**：同一版本每次冷启动最多弹出一次

**前置条件**：
- 已经历过一次冷启动并看到了更新对话框（点击了「稍后提醒」）
- 再次完全退出 App 后重新冷启动

**操作步骤**：
1. 上一次会话点击「稍后提醒」后退出 App
2. 重新冷启动 App

1. 新的冷启动会话中，若远端版本仍高于本地，更新对话框应重新弹出（内存标记不跨进程持久化）

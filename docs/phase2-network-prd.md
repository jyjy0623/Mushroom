## 一、背景与目标

### 1.1 背景

Phase 1 完成了应用的完整本地功能，包括任务打卡、蘑菇积分、里程碑、奖励解锁等核心玩法。所有数据存储于设备本地 Room 数据库，用户更换设备或重装 App 后数据会丢失，且无法跨设备同步。

Phase 2 的目标是为应用引入网络能力，使数据更安全、玩法更有趣。

### 1.2 核心目标

1. **云端备份**：用户数据可上传至远端服务器，更换设备或误删后可恢复
2. **联网排行榜**：游戏积分可与其他用户比较，增强竞技趣味性

### 1.3 非目标（Phase 2 不做）

- 实时多人协作或家庭组功能
- 社交关注 / 好友系统
- 内容分享（任务模板共享等）
- 服务端推送消息（Push Notification 联网版）

## 三、功能需求

### 3.1 用户身份与账号

#### 3.1.1 账号注册 / 登录

| ID | 需求 |
|----|------|
| AUTH-01 | 用户可以用**手机号 + 短信验证码**注册账号 |
| AUTH-02 | 用户可以用手机号 + 验证码登录 |
| AUTH-03 | 登录状态持久化，App 重启后无需重新登录（Token 本地存储） |
| AUTH-04 | 用户可以退出登录，退出后本地数据保留，云端数据不删除 |
| AUTH-05 | 每个设备可绑定一个账号；一个账号可在最多 2 台设备使用 |

> **说明**：Phase 2 不做 OAuth（微信/Google），降低实现复杂度。

#### 3.1.2 账号与设备

| ID | 需求 |
|----|------|
| AUTH-10 | 账号包含一个"孩子档案"（名称、头像），登录后自动加载 |
| AUTH-11 | 用户（孩子）可直接查看并编辑账号信息（昵称、头像） |

### 3.2 云端备份与恢复

#### 3.2.1 备份触发

| ID | 需求 |
|----|------|
| BACKUP-01 | 用户已登录时，App 每天**自动备份一次**（后台静默，不打扰用户） |
| BACKUP-02 | 用户可在设置页手动触发"立即备份" |
| BACKUP-03 | 备份内容包含：任务、打卡记录、蘑菇账本、里程碑、奖励、扣分记录、模板配置 |
| BACKUP-04 | 备份不包含：设备本地的图片附件（阶段二暂不支持图片云同步） |
| BACKUP-05 | 备份采用**增量方式**：记录上次备份时间戳，仅上传新增/变更数据 |
| BACKUP-06 | 未登录时，备份功能不可用，设置页入口显示"需要登录"提示 |

#### 3.2.2 备份状态展示

| ID | 需求 |
|----|------|
| BACKUP-10 | 设置页显示"上次成功备份时间" |
| BACKUP-11 | 备份进行中时显示 loading 状态 |
| BACKUP-12 | 备份失败时显示错误原因（网络不可用 / 服务器错误），可重试 |

#### 3.2.3 数据恢复

| ID | 需求 |
|----|------|
| RESTORE-01 | 用户登录新设备后，设置页提供"从云端恢复数据"入口 |
| RESTORE-02 | 恢复前显示云端备份的时间和数据概览（任务数、打卡天数等） |
| RESTORE-03 | 恢复操作**覆盖本地数据**，须弹出确认对话框（二次确认防误操作） |
| RESTORE-04 | 恢复成功后 App 重新加载数据 |
| RESTORE-05 | 恢复失败时提示错误，本地数据不受影响 |

## 四、非功能需求

| 类别 | 要求 |
|------|------|
| **安全** | 所有网络请求走 HTTPS；用户密码不存本地，Token 存 EncryptedSharedPreferences |
| **隐私** | 备份数据在服务端加密存储；排行榜昵称可自定义（不暴露真实信息） |
| **离线** | 无网络时 App 所有本地功能正常使用，网络功能降级处理（提示不可用） |
| **性能** | 备份/恢复不阻塞主线程；排行榜首屏加载 ≤ 2s（良好网络） |
| **兼容性** | minSdk 26（Android 8.0+），与现有功能无缝共存 |

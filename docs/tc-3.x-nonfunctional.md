## 3.1 性能

---

#### TC-3.1-01

**用例名称**：App 冷启动时间 < 2 秒

**前置条件**：
- App 已完全退出（非后台）
- 设备性能正常（非调试构建限速状态）

**操作步骤**：
1. 记录点击 App 图标的时刻（T0）
2. 观察主界面（每日任务列表）完全显示的时刻（T1）

**预期结果**：
1. T1 - T0 < 2000ms（2 秒）
2. 启动期间无长时间白屏

- [ ] 通过

---

#### TC-3.1-02

**用例名称**：底部导航切换页面流畅（60fps），无明显卡顿

**前置条件**：
- App 已启动并处于主界面
- 存在一定量的数据（任务、蘑菇、统计）

**操作步骤**：
1. 连续快速点击底部导航栏：任务 → 蘑菇账本 → 奖品 → 统计 → 设置 → 任务（循环 3 次）
2. 观察每次页面切换是否流畅

**预期结果**：
1. 每次页面切换动画流畅，无明显跳帧（肉眼无卡顿感）
2. 不出现 ANR（无响应对话框）
3. 页面内容在切换后正确加载显示

- [ ] 通过

---

#### TC-3.1-03

**用例名称**：统计页各 Tab 切换流畅，无卡顿

**前置条件**：
- 统计页数据已加载完成
- 数据量适中（约 30 天打卡记录）

**操作步骤**：
1. 在统计页反复快速切换「学习情况」→「蘑菇收支」→「成绩趋势」各 5 次
2. 同时多次切换周期筛选（近7天 / 近30天 / 本学期）

**预期结果**：
1. Tab 切换响应迅速，无明显延迟
2. 周期切换后数据刷新不出现白屏或闪烁
3. 全程 App 不崩溃

- [ ] 通过

---

#### TC-3.1-04

**用例名称**：奖品列表滚动流畅

**前置条件**：
- 奖品列表中有 10+ 个奖品

**操作步骤**：
1. 进入奖品页
2. 快速上下滑动奖品卡片列表

**预期结果**：
1. 滑动流畅，无明显卡顿
2. 奖品卡片中的进度条和文字随滚动正确渲染

- [ ] 通过

## 3.3 安全

---

#### TC-3.3-01

**用例名称**：家长 PIN 验证——兑换奖品需要家长授权

**前置条件**：
- 家长已设置 PIN（如 1234）
- 学生蘑菇余额充足

**操作步骤**：
1. 进入任一奖品详情页（如「看电视」）
2. 设置兑换参数，点击「确认兑换」
3. 观察是否弹出 PIN 验证对话框
4. 不输入 PIN，直接点击「确认」按钮（此时按钮应禁用）
5. 输入任意非数字字符（如字母）
6. 输入正确 PIN 后确认

**预期结果**：
1. 点击「确认兑换」后，弹出「家长验证」对话框
2. 对话框描述文字明确说明「兑换奖品需要家长授权」
3. PIN 输入框为空时，「确认」按钮处于禁用状态（灰色），不可点击
4. 输入非数字字符时，输入框不接受，内容不变（仅接受数字，最多 6 位）
5. 输入正确 PIN 后，兑换成功；未验证时无法完成兑换

- [ ] 通过

---

#### TC-3.3-02

**用例名称**：家长 PIN 验证——成绩录入需要家长授权

**前置条件**：
- 家长 PIN 已设置

**操作步骤**：
1. 进入里程碑页面，尝试录入一条成绩
2. 观察是否弹出 PIN 验证对话框
3. 输入正确 PIN 确认

**预期结果**：
1. 录入成绩操作触发「家长验证」对话框
2. 描述文字为「录入成绩需要家长授权，请输入家长 PIN 码。」
3. 正确 PIN 输入后，成绩录入成功
4. 未验证时，成绩不被录入

- [ ] 通过

---

#### TC-3.3-03

**用例名称**：家长 PIN 验证——进入设置页需要验证

**前置条件**：
- 家长 PIN 已设置

**操作步骤**：
1. 点击底部导航进入「设置」页
2. 观察是否弹出 PIN 验证对话框
3. 输入正确 PIN 确认

**预期结果**：
1. 进入设置时触发「家长验证」对话框
2. 描述文字为「修改家长设置需要验证，请输入家长 PIN 码。」
3. 正确 PIN 输入后，进入设置页
4. 取消（点击「取消」）后，不进入设置页

- [ ] 通过

---

#### TC-3.3-04

**用例名称**：PIN 输入框密码掩码显示

**前置条件**：
- 家长 PIN 验证对话框已弹出

**操作步骤**：
1. 在 PIN 输入框中输入数字（如 1234）
2. 观察输入框显示内容

**预期结果**：
1. 输入内容以密码掩码（圆点 ●）显示，不以明文显示
2. 软键盘弹出数字键盘（KeyboardType.NumberPassword），非全键盘
3. 最多只能输入 6 位数字

- [ ] 通过

---

#### TC-3.3-05

**用例名称**：PIN 错误后显示错误提示且不自动关闭对话框

**前置条件**：
- 家长 PIN 为 1234，验证对话框已弹出

**操作步骤**：
1. 输入错误 PIN（如 9999），点击「确认」
2. 观察对话框状态

**预期结果**：
1. 对话框保持打开状态，不自动关闭
2. PIN 输入框下方显示红色错误文字「PIN 错误，请重试」
3. 输入框边框变为红色（isError = true 状态）
4. 可继续尝试输入正确 PIN

- [ ] 通过

---

#### TC-3.3-06

**用例名称**：家长 PIN 验证对话框弹出时自动获取焦点

**前置条件**：
- 家长 PIN 验证对话框已弹出

**操作步骤**：
1. 观察对话框弹出后，不手动点击输入框的情况下软键盘状态

**预期结果**：
1. 对话框弹出后，PIN 输入框自动获取焦点（FocusRequester 触发）
2. 软键盘自动弹起，用户无需额外点击即可直接输入

- [ ] 通过

## 3.4 易用性

---

#### TC-3.4-01

**用例名称**：界面语言为中文——所有页面文字均为中文

**前置条件**：
- App 已安装并运行

**操作步骤**：
1. 逐一访问所有主要页面：任务列表、蘑菇账本、奖品列表、奖品详情、统计、设置
2. 观察所有 UI 文字

**预期结果**：
1. 所有按钮、标签、标题、提示文字均为中文
2. 无未翻译的英文枚举值（如不出现 MINI_TEST、EARLY_DONE 等原始枚举名）
3. 错误提示、空态提示、Snackbar 均为中文

- [ ] 通过

---

#### TC-3.4-02

**用例名称**：蘑菇图标视觉——各等级蘑菇图标正确显示

**前置条件**：
- 学生拥有各等级蘑菇余额

**操作步骤**：
1. 进入蘑菇账本页，观察各等级蘑菇图标
2. 进入统计页「蘑菇收支」Tab，观察余额卡片中的图标

**预期结果**：
1. 小蘑菇显示 🍄
2. 中蘑菇显示 🍄‍🟫
3. 大蘑菇显示 🌟
4. 金蘑菇显示 ✨
5. 传说蘑菇显示 👑
6. 各图标清晰可见，尺寸适当，在深色/浅色主题下均正常显示

- [ ] 通过

---

#### TC-3.4-03

**用例名称**：拼图元素视觉——解锁块与未解锁块视觉区分明显

**前置条件**：
- 高达模型奖品已解锁 10/20 块

**操作步骤**：
1. 进入高达模型奖品详情页
2. 观察拼图格子区域

**预期结果**：
1. 已解锁的 10 块格子显示为主题色背景，内含 🍄 图标，视觉醒目可爱
2. 未解锁的 10 块格子显示为灰色（surfaceVariant）占位块，与已解锁块颜色区分明显
3. 格子尺寸统一（32dp × 32dp），排列整齐
4. 整体拼图区域视觉风格可爱，符合面向儿童的产品定位

- [ ] 通过

---

#### TC-3.4-04

**用例名称**：关键 UI 元素字体大小——重要数字清晰大字显示

**前置条件**：
- 学生有打卡连续记录和蘑菇余额

**操作步骤**：
1. 进入统计页「学习情况」Tab，观察当前连续天数数字
2. 观察「平均日完成率」百分比数字
3. 进入「蘑菇收支」Tab，观察各等级余额数字

**预期结果**：
1. Streak 卡片中连续天数数字使用 titleLarge 样式（大字体，突出显示）
2. 完成率百分比使用 headlineMedium 样式（更大字体）
3. 余额数字粗体（FontWeight.Bold）显示
4. 所有关键数字在不放大字体的情况下清晰易读

- [ ] 通过

#### TC-3.4-05

**用例名称**：空态提示友好——首次使用时各页面空态显示合适文字

**前置条件**：
- 全新账户，无任何数据

**操作步骤**：
1. 进入奖品列表，观察空态显示
2. 进入统计页各 Tab，观察空态显示
3. 成绩趋势 Tab 中观察无成绩时的空态

**预期结果**：
1. 奖品列表空态显示「还没有奖品，请让家长创建」，文字友好，指引清晰
2. 打卡统计空态显示「暂无打卡数据」
3. 蘑菇统计空态显示「暂无蘑菇数据」
4. 成绩趋势空态显示「XX暂无成绩记录」（包含所选学科名称）
5. 所有空态文字使用柔和的次级色（onSurfaceVariant），不使用红色或警告色

#### TC-3.4-06

**用例名称**：庆祝动画视觉——任务全部完成时庆祝横幅出现与消失

**前置条件**：
- 当日任务尚有 1 条未完成

**操作步骤**：
1. 完成最后一条任务，触发全勤
2. 观察任务列表顶部庆祝横幅的出现动画和消失时机

**预期结果**：
1. 全勤触发后，庆祝横幅通过 AnimatedVisibility（缩放+淡入）动画出现
2. 横幅视觉可爱，内容积极（如「🎉 全部完成！」）
3. 约 3 秒后，横幅自动消失
4. 横幅出现和消失过程流畅，无突然闪现/消失

- [ ] 通过

---

#### TC-3.4-07

**用例名称**：任务完成勾选动画——打卡成功时弹入动画流畅

**前置条件**：
- 任务列表中有未完成的任务

**操作步骤**：
1. 点击某个任务的「完成打卡」按钮
2. 观察打卡确认后任务卡片的状态变化动画

**预期结果**：
1. 打卡成功后，任务卡片上的完成勾选标记通过 animateFloatAsState 缩放弹入（约 180ms）
2. 动画效果自然，有弹性感（scale 从 0 到 1.2 再到 1.0）
3. 动画期间不阻塞用户操作其他任务

- [ ] 通过

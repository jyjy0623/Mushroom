## 2.8.1 页面入口与 Tab 切换

---

#### TC-2.8.1-01

**用例名称**：通过底部导航进入统计页

**前置条件**：
- App 已启动，处于任意主界面（任务/奖品/蘑菇等页面）

**操作步骤**：
1. 点击底部导航栏中的「统计」图标
2. 观察页面加载状态

**预期结果**：
1. 页面跳转到统计页，顶部标题显示「数据统计」
2. 页面正常加载，不发生崩溃
3. 可见三个 Tab：「学习情况」、「蘑菇收支」、「成绩趋势」（对应需求中的打卡/蘑菇/成绩）
4. 可见周期筛选区域：「近7天」「近30天」「本学期」三个芯片（FilterChip）
5. 默认选中「近7天」周期，默认激活「学习情况」Tab（Tab index 0）

- [ ] 通过

---

#### TC-2.8.1-02

**用例名称**：Tab 切换——点击三个 Tab 均正常响应

**前置条件**：
- 已进入统计页，存在一定历史数据

**操作步骤**：
1. 点击「蘑菇收支」Tab
2. 观察内容区域变化
3. 点击「成绩趋势」Tab
4. 观察内容区域变化
5. 点击「学习情况」Tab 返回
6. 观察内容区域变化

**预期结果**：
1. 点击「蘑菇收支」Tab 后，Tab 下划线移动至该 Tab，内容区域显示蘑菇相关统计（当前余额、收支卡片）
2. 点击「成绩趋势」Tab 后，内容区域显示学科选择行和成绩统计卡片
3. 点击「学习情况」Tab 后，内容区域恢复显示打卡连续天数、完成率等内容
4. 三次切换过程中页面均不崩溃

- [ ] 通过

---

#### TC-2.8.1-03

**用例名称**：数据加载中显示 Loading 状态

**前置条件**：
- 统计数据量较大，加载可能有短暂延迟（可通过网络限速或大量数据模拟）

**操作步骤**：
1. 进入统计页，在数据加载完成前观察页面状态

**预期结果**：
1. 加载期间，内容区域显示「加载中…」文字（居中显示）
2. 加载完成后，文字消失，正常内容显示
3. 加载过程中 Tab 和周期筛选区域仍可见（骨架布局保持稳定）

- [ ] 通过

## 2.8.2 统计周期筛选

---

#### TC-2.8.2-01

**用例名称**：切换统计周期——近7天 / 近30天 / 本学期

**前置条件**：
- 统计页已加载，存在跨不同时间段的历史打卡数据

**操作步骤**：
1. 确认默认周期芯片「近7天」已选中（填充样式）
2. 点击「近30天」芯片
3. 观察数据变化
4. 点击「本学期」芯片
5. 观察数据变化
6. 再点击「近7天」切回

**预期结果**：
1. 初始状态「近7天」芯片为选中态（filled），其余为未选中态（outlined）
2. 点击「近30天」后，该芯片切换为选中态，「近7天」变为未选中
3. 内容区域的统计数据（连续天数、完成率等）刷新，数字反映 30 天范围的统计
4. 点击「本学期」后同样刷新，数据范围扩展到本学期起始日
5. 回切「近7天」后数据恢复为 7 天范围的统计
6. 三次切换均不崩溃

- [ ] 通过

---

#### TC-2.8.2-02

**用例名称**：切换周期后各 Tab 数据均更新

**前置条件**：
- 统计页已加载，存在不同周期数据差异

**操作步骤**：
1. 在「学习情况」Tab 下切换到「近30天」周期，记录连续天数数字
2. 点击「蘑菇收支」Tab，观察收支数字
3. 点击「成绩趋势」Tab，观察成绩列表
4. 切换回「近7天」，分别检查三个 Tab

**预期结果**：
1. 各 Tab 均在周期切换后刷新数据，不存在某个 Tab 仍显示上一周期数据的情况
2. 不同周期下数据数值合理（30 天数字通常 ≥ 7 天数字）

- [ ] 通过

## 2.8.3 打卡统计 Tab

---

#### TC-2.8.3-01

**用例名称**：正向流程——打卡统计显示连续天数与完成率

**前置条件**：
- 学生近 7 天内有打卡记录：连续打卡 5 天，最长连续 12 天（历史），总打卡 35 次
- 近 7 天平均完成率约 80%

**操作步骤**：
1. 进入统计页，选择「近7天」周期
2. 点击「学习情况」Tab
3. 观察 Streak 卡片和完成率卡片

**预期结果**：
1. Streak 卡片中，「当前连续」显示 5，「最长连续」显示 12，「总打卡」显示 35
2. 「平均日完成率」卡片显示「80%」（大字体，主题色）
3. 完成率进度条显示约 80% 填充
4. 所有数字与实际数据库一致

- [ ] 通过

---

#### TC-2.8.3-02

**用例名称**：学科打卡比例——按学科展示完成率

**前置条件**：
- 近 7 天内有数学 100%、语文 80%、英语 60% 的打卡完成率记录
- 其他学科无打卡任务（比例为 0，不应显示）

**操作步骤**：
1. 进入「学习情况」Tab，选择「近7天」周期
2. 下滑到「学科完成率」卡片

**预期结果**：
1. 显示数学、语文、英语三个学科的进度条
2. 数学显示「数学 100%」，进度条满格
3. 语文显示「语文 80%」，进度条约 80%
4. 英语显示「英语 60%」，进度条约 60%
5. 其他无打卡任务的学科不显示（过滤 rate > 0 的才显示）

- [ ] 通过

---

#### TC-2.8.3-03

**用例名称**：边界值——无任何打卡数据时显示空态

**前置条件**：

**操作步骤**：
1. 进入统计页，选择「近7天」周期
2. 点击「学习情况」Tab

**预期结果**：
1. 内容区域显示「暂无打卡数据」文字（居中）
2. 不显示空的卡片或乱码数字
3. 不崩溃

- [ ] 通过

#### TC-2.8.3-04

**用例名称**：连续天数 streak 归零后显示为 0

**前置条件**：

1. 「当前连续」显示 0 天
2. 「最长连续」仍显示历史最长连续天数（不归零）

4. 各等级蘑菇图标与名称对应正确（🍄=小蘑菇，🍄‍🟫=中蘑菇，🌟=大蘑菇）

## 2.8.5 成绩统计 Tab

---

#### TC-2.8.5-01

**用例名称**：正向流程——选择数学学科查看成绩统计

**前置条件**：
- 数学有 3 条成绩记录：小测 92 分、周测 85 分、月考 78 分
- 其他学科无成绩记录

**操作步骤**：
1. 进入统计页，点击「成绩趋势」Tab
2. 确认学科选择行默认选中「数学」
3. 观察成绩统计卡片和成绩列表

**预期结果**：
1. 学科选择行默认选中「数学」（filled 状态）
2. 成绩统计卡片显示：平均分 85（(92+85+78)/3≈85），最高分 92
3. 趋势显示「需关注 ↓」或「稳定」（根据数据计算结果）
4. 成绩列表按时间倒序展示 3 条记录，每条包含名称、日期、类型、分数
5. 92 分以绿色（tertiary）显示，78 分以主题色显示（≥60 且 <90）

- [ ] 通过

---

#### TC-2.8.5-02

**用例名称**：切换学科——点击不同学科芯片切换成绩数据

**前置条件**：
- 数学有 3 条成绩，英语有 2 条成绩，语文无成绩

**操作步骤**：
1. 进入「成绩趋势」Tab，确认默认显示数学
2. 点击「英语」芯片
3. 观察数据变化
4. 点击「语文」芯片
5. 观察数据变化

**预期结果**：
1. 点击「英语」后，芯片切换到选中状态，成绩统计更新为英语的 2 条数据
2. 点击「语文」后，内容区域显示「语文暂无成绩记录」（空态提示）
3. 空态显示不崩溃，文字正确包含所选学科名（如「语文暂无成绩记录」）

- [ ] 通过

---

#### TC-2.8.5-03

**用例名称**：成绩趋势标识——进步/稳定/需关注显示正确颜色

**前置条件**：
- 数学最近 3 次成绩为 75、85、92（上升趋势）
- 英语最近 3 次成绩为 80、80、80（稳定）
- 物理最近 3 次成绩为 90、75、60（下降趋势）

**操作步骤**：
1. 在「成绩趋势」Tab，分别点击「数学」「英语」「物理」芯片
2. 各自观察趋势文字和颜色

**预期结果**：
1. 数学：趋势显示「进步中 ↑」，文字颜色为绿色（tertiary）
2. 英语：趋势显示「稳定」，文字颜色为主题色（primary）
3. 物理：趋势显示「需关注 ↓」，文字颜色为红色（error）

- [ ] 通过

---

#### TC-2.8.5-04

**用例名称**：边界值——低于 60 分的成绩分数颜色为红色

**前置条件**：
